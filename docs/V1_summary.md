# V1 阶段总结

## 1. 项目概述

- 项目目标：面向纯回转体工件的水浸自动化超声检测仿形路径规划，完成 STEP 模型到扫描路径与 CSV 导出的 V1 主链路。
- 技术栈：Python + OCP + PyQt6。

## 2. V1 实现范围

- STEP 读取：支持 OCP 优先读取，并保留文本解析 fallback。
- 标准化：采用 V1 保守策略，对简单纯回转体进行标准化处理。
- 母线提取：基于 `Y=0` 截面提取外轮廓母线点列。
- 路径生成：基于离散母线生成逐层扫描路径。
- CSV 导出：支持标准版与精简版导出。
- CLI 调试工具：提供 OCP 导入、STEP 读取、母线提取、路径联调脚本。
- PyQt6 GUI 原型：完成最小桌面界面，支持加载 STEP、提取母线、生成路径、导出 CSV。

## 3. 核心架构

- `core/`：STEP 读取、标准化、母线提取、路径规划等核心逻辑。
- `data/`：核心数据结构定义，如 `ScanParams`、`PathPoint`、`ScanPath`、STEP 相关模型。
- `gui/`：PyQt6 原型界面与控制器层。
- `scripts/`：调试、验证、fixture 生成与 GUI 启动脚本。
- `tests/`：路径规划、OCP 链路、圆柱/圆台/圆角件验证测试。

设计思想是“薄 GUI + 厚 core”：

- GUI 只负责参数输入、状态展示与操作编排。
- 几何处理、路径生成和导出逻辑统一放在 `core/` 与相关模块中。
- 这样便于后续继续扩展 CLI、自动化测试和更完整的桌面界面，而不重写算法链路。

## 4. 已验证能力

- 圆柱：标准件路径角度可验证为约 `90°`。
- 圆台：法向方向问题已分析并形成一致结论，能够区分“几何法向”与“工艺语义方向”。
- 圆角连续性：圆角过渡件路径角度连续，无明显跳变。
- OCP 链路打通：已完成 `STEP -> OCP shape -> 标准化 -> 截面母线 -> 路径生成` 主链路验证。

## 5. 当前已知限制

- 坐标系方向依赖 STEP 模型原始姿态。
- Z 轴方向可能与工艺定义不一致。
- 母线方向语义未完全统一。
- 当前不自动修正旋转轴方向。
- `profile_extractor` 对复杂件仍有限制。

补充说明：

- V1 主要优先支持简单纯回转体与标准姿态模型。
- 对复杂特征、多轮廓、强噪声截面，目前仍采用保守策略。

## 6. 本轮关键问题结论

👉 圆台出现 `116°` 不是算法错误  
👉 是母线方向 + 外法向定义共同决定的结果  
👉 V1 不做自动纠正

说明：

- 当前 `theta = atan2(nx, nz)` 的定义保持不变。
- 当母线方向与工艺理解中的“上大下小”或“上小下大”语义不一致时，几何外法向仍然可能落入不同象限。
- 这一问题在 V1 中只做结论澄清，不在核心算法里做自动语义修正。

## 7. V2 规划

未来将增加：

- 模型预览（2D/3D）
- 旋转轴显示
- 坐标系可视化
- 手动翻转 Z 轴
- 母线方向修正
- 路径预览

预期目标：

- 让几何结果与工艺语义在界面层可见、可控、可修正。
- 在不破坏核心算法稳定性的前提下，提高复杂真实模型的可解释性与可操作性。
